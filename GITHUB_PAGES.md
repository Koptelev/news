# Настройка GitHub Pages

## Как настроить GitHub Pages для отображения веб-интерфейса

1. Перейдите в настройки репозитория на GitHub: https://github.com/Koptelev/news/settings/pages
2. Откройте раздел **Pages** (в меню слева)
3. В разделе **Source** выберите:
   - **Branch**: `main`
   - **Folder**: `/ (root)` - используйте корневую директорию
4. Нажмите **Save**

После этого GitHub Pages будет использовать файл `index.html` из корня репозитория.

**Альтернативный вариант:** Если хотите использовать папку `/docs`, выберите **Folder**: `/docs` в настройках.

## Важно

GitHub Pages может отображать только статические файлы. Для полной функциональности сервиса необходимо:

1. Запустить локальный сервер:
```bash
uvicorn src.main:app --reload
```

2. Открыть `http://localhost:8000` в браузере

Страница на GitHub Pages будет показывать инструкции по запуску и интерфейс, который автоматически подключится к локальному серверу, если он запущен.

## Альтернативные варианты деплоя

Для полноценного деплоя сервиса можно использовать:

- **Vercel** - https://vercel.com
- **Railway** - https://railway.app
- **Render** - https://render.com
- **Fly.io** - https://fly.io

Все эти платформы поддерживают Python/FastAPI приложения.
